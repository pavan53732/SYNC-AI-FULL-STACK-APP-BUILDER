## 1. Overview

Every application generated by Sync AI follows a deterministic solution structure based on the target framework. The structure is determined by `targetPlatform.frameworkFamily` in the Structured Spec.

### Framework Strategy Matrix

| Framework | Solution Structure | Pattern | Packaging | Database |
| --------- | ------------------ | ------- | --------- | -------- |
| **WinUI3** | App/Core/Data | MVVM | MSIX | SQLite/EF Core |
| **WPF** | App/Core/Data | MVVM | MSI/MSIX | SQLite/EF Core |
| **WinForms** | App (single) | Code-Behind | MSI | SQLite/EF Core |
| **Console** | App (single) | N/A | EXE | Optional |
| **Win32** | Single .vcxproj | Procedural | EXE | Manual |
| **WinRT** | App/WinRT | MVVM | MSIX | SQLite |
| **Hybrid** | App + NativeLib | MVVM + Procedural | MSIX/EXE | SQLite/EF Core |

### Architecture Philosophy (WinUI 3 / WPF / WinRT)

| Aspect                   | Rule                                            |
| ------------------------ | ----------------------------------------------- |
| **Pattern**              | MVVM (Model-View-ViewModel)                     |
| **Separation**           | Strict: UI knows nothing about data access      |
| **Testability**          | Core and Data projects have no WinUI dependency |
| **Dependency Direction** | App → Core ← Data                               |
| **DI Container**         | `Microsoft.Extensions.DependencyInjection`      |
| **Database**             | SQLite via EF Core 8.x                          |
| **Packaging**            | MSIX (self-contained)                           |

## 3. Project: App (WinUI 3 Host)

### .csproj Template

````xml
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>WinExe</OutputType>
    <TargetFramework>net8.0-windows10.0.22621.0</TargetFramework>
    <TargetPlatformMinVersion>10.0.22621.0</TargetPlatformMinVersion>
    <RootNamespace>{AppName}</RootNamespace>
    <ApplicationManifest>Package.appxmanifest</ApplicationManifest>
    <Platforms>x86;x64;arm64</Platforms>
    <RuntimeIdentifiers>win-x86;win-x64;win-arm64</RuntimeIdentifiers>
    <PublishProfile>Properties/PublishProfiles/win-$(Platform).pubxml</PublishProfile>
    <UseWinUI>true</UseWinUI>
    <WindowsAppSdkSelfContained>true</WindowsAppSdkSelfContained>
    <WindowsPackageType>MSIX</WindowsPackageType>
    <Nullable>enable</Nullable>
    <ImplicitUsings>enable</ImplicitUsings>
    <!-- NuGet Lock-File Policy: Ensures transitive dependency resolution is identical on every restore -->
    <RestorePackagesWithLockFile>true</RestorePackagesWithLockFile>
    <RestoreLockedMode>true</RestoreLockedMode>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Microsoft.WindowsAppSDK" Version="1.5.250211001" />
    <PackageReference Include="Microsoft.Windows.SDK.BuildTools" Version="10.0.22621.756" />
    <PackageReference Include="CommunityToolkit.WinUI.UI.Controls" Version="7.1.2" />
  </ItemGroup>

  <ItemGroup>
    <ProjectReference Include="..\{AppName}.Core\{AppName}.Core.csproj" />
    <ProjectReference Include="..\{AppName}.Data\{AppName}.Data.csproj" />
  </ItemGroup>
</Project>
```

> **NuGet Lock-File Policy**: Every generated project commits `packages.lock.json` to the workspace. During release builds, MSBuild is invoked with `/p:RestoreLockedMode=true` to enforce exact package versions and detect any drift from the recorded lock file.
> If a lock file is absent or stale, the build fails with `NETSDK1138` and the system automatically regenerates it before retrying.
> See [TOOLCHAIN_MANIFEST.md](./TOOLCHAIN_MANIFEST.md) §7 for redistribution terms covering NuGet packages.

### Cross-Compilation & VC++ Toolchain Dependency

While Sync AI generates C# code, the Windows App SDK (WinUI 3) fundamentally relies on native C++ components.
To satisfy the `<Platforms>` and `<RuntimeIdentifiers>` targets, the bundled toolchain MUST provide cross-compilation support:

- **VC++ Toolchain:** The build environment relies on MSBuild and the Windows SDK to compile/link any native shim dependencies required by `Microsoft.WindowsAppSDK`.
- **ARM64 Strategy:** By defining `win-arm64` in the `RuntimeIdentifiers`, the `dotnet publish` command explicitly outputs an ARM64 native binary package. The bundled toolchain includes the necessary targeting packs and SDK libraries to perform this cross-compilation natively on an x64 host.
````

### main.cpp Entry Point Pattern

```cpp
#include "main.h"
#include "MainWindow.h"

#define MAX_LOADSTRING 100

// Global variables
HINSTANCE hInst;                                // current instance
WCHAR szTitle[MAX_LOADSTRING];                  // The title bar text
WCHAR szWindowClass[MAX_LOADSTRING];            // the main window class name

// Forward declarations
ATOM                MyRegisterClass(HINSTANCE hInstance);
BOOL                InitInstance(HINSTANCE, int);
LRESULT CALLBACK    WndProc(HWND, UINT, WPARAM, LPARAM);

int APIENTRY wWinMain(_In_ HINSTANCE hInstance,
                      _In_opt_ HINSTANCE hPrevInstance,
                      _In_ LPWSTR    lpCmdLine,
                      _In_ int       nCmdShow)
{
    UNREFERENCED_PARAMETER(hPrevInstance);
    UNREFERENCED_PARAMETER(lpCmdLine);

    // Initialize global strings
    LoadStringW(hInstance, IDC_APPNAME, szWindowClass, MAX_LOADSTRING);
    MyRegisterClass(hInstance);

    // Perform application initialization
    if (!InitInstance(hInstance, nCmdShow))
    {
        return FALSE;
    }

    HACCEL hAccelTable = LoadAccelerators(hInstance, MAKEINTRESOURCE(IDC_APPNAME));
    MSG msg;

    // Main message loop
    while (GetMessage(&msg, nullptr, 0, 0))
    {
        if (!TranslateAccelerator(msg.hwnd, hAccelTable, &msg))
        {
            TranslateMessage(&msg);
            DispatchMessage(&msg);
        }
    }

    return (int)msg.wParam;
}
```

### Window Procedure Pattern

```cpp
// MainWindow.cpp
#include "MainWindow.h"

LRESULT CALLBACK WndProc(HWND hWnd, UINT message, WPARAM wParam, LPARAM lParam)
{
    switch (message)
    {
    case WM_CREATE:
        // Initialize window resources
        return 0;
        
    case WM_PAINT:
        {
            PAINTSTRUCT ps;
            HDC hdc = BeginPaint(hWnd, &ps);
            FillRect(hdc, &ps.rcPaint, (HBRUSH)(COLOR_WINDOW + 1));
            EndPaint(hWnd, &ps);
        }
        break;
        
    case WM_DESTROY:
        PostQuitMessage(0);
        break;
        
    default:
        return DefWindowProc(hWnd, message, wParam, lParam);
    }
    return 0;
}
```

### Repository Pattern

```csharp
namespace {AppName}.Data.Repositories;

public class {EntityName}Repository : I{EntityName}Repository
{
    private readonly AppDbContext _context;

    public {EntityName}Repository(AppDbContext context)
    {
        _context = context;
    }

    public async Task<IEnumerable<{EntityName}>> GetAllAsync()
        => await _context.{EntityName}s.AsNoTracking().ToListAsync();

    public async Task<{EntityName}?> GetByIdAsync(Guid id)
        => await _context.{EntityName}s.FindAsync(id);

    public async Task AddAsync({EntityName} entity)
    {
        _context.{EntityName}s.Add(entity);
        await _context.SaveChangesAsync();
    }

    public async Task UpdateAsync({EntityName} entity)
    {
        _context.{EntityName}s.Update(entity);
        await _context.SaveChangesAsync();
    }

    public async Task DeleteAsync(Guid id)
    {
        var entity = await GetByIdAsync(id);
        if (entity != null)
        {
            _context.{EntityName}s.Remove(entity);
            await _context.SaveChangesAsync();
        }
    }
}
```

## 6. Dependency Rules

```text
{AppName}.App
    ├── Depends on: {AppName}.Core
    ├── Depends on: {AppName}.Data
    └── Knows about: All layers (wires DI)

{AppName}.Core
    ├── Depends on: nothing (pure C#)
    └── Defines: Models, Service interfaces, Service implementations

{AppName}.Data
    ├── Depends on: {AppName}.Core (for model types)
    ├── Depends on: EF Core + SQLite packages
    └── Defines: AppDbContext, Repositories
```

### Forbidden Dependencies

| Project | Forbidden Dependency          | Reason                                       |
| ------- | ----------------------------- | -------------------------------------------- |
| `Core`  | WinUI 3 / Windows App SDK     | Business logic must be platform-neutral      |
| `Core`  | EF Core                       | Core must not know about data persistence    |
| `Data`  | WinUI 3 / Windows App SDK     | Data access must be platform-neutral         |
| `App`   | Direct SQL / raw SQLite calls | All data access must go through repositories |

## 8. Dependency Injection Bootstrap

All DI registration happens in `App.xaml.cs`:

```csharp
private static IServiceProvider ConfigureServices()
{
    var services = new ServiceCollection();

    // Database
    var dbPath = Path.Combine(
        Environment.GetFolderPath(Environment.SpecialFolder.LocalApplicationData),
        "{AppName}", "app.db");
    Directory.CreateDirectory(Path.GetDirectoryName(dbPath)!);
    services.AddDbContext<AppDbContext>(options =>
        options.UseSqlite($"Data Source={dbPath}"));

    // Repositories (one per entity)
    services.AddScoped<I{Entity1}Repository, {Entity1}Repository>();
    services.AddScoped<I{Entity2}Repository, {Entity2}Repository>();

    // Services (one per feature group)
    services.AddScoped<I{Feature1}Service, {Feature1}Service>();
    services.AddScoped<I{Feature2}Service, {Feature2}Service>();

    // ViewModels (one per page/dialog)
    services.AddTransient<{Page1}ViewModel>();
    services.AddTransient<{Page2}ViewModel>();
    services.AddTransient<{Dialog1}ViewModel>();

    return services.BuildServiceProvider();
}
```

### Service Resolution Helper

```csharp
// In App.xaml.cs
private static IServiceProvider? _serviceProvider;

public static T GetService<T>() where T : class
    => _serviceProvider?.GetRequiredService<T>()
       ?? throw new InvalidOperationException("Service provider not initialized.");
```

## 11. Base Project Template (Minimal Kernel Bootstrap)

The Minimal Kernel Bootstrap provides the **empty scaffolding** that the Architect Agent starts with. It contains only the shell files — no business logic, no entities, no pages beyond the empty main window.

### Bootstrap File Set

| File                   | Purpose                                   |
| ---------------------- | ----------------------------------------- |
| `{AppName}.App.csproj` | Valid .NET 8 WinUI 3 project file         |
| `App.xaml`             | WinUI 3 Application root                  |
| `App.xaml.cs`          | Application entry point (empty handlers)  |
| `MainWindow.xaml`      | Empty shell window                        |
| `MainWindow.xaml.cs`   | Window code-behind skeleton               |
| `Package.appxmanifest` | Skeleton manifest (publisher/version TBD) |

> **CRITICAL**: The bootstrap provides ONLY empty scaffolding. All business logic, entities, pages, services, and repositories are generated by AI agents from the `AppSpec`. The bootstrap itself contains zero business logic.
